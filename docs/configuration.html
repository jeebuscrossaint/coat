Configuration

Everything is controlled from ~/.config/coat/coat.yaml.

Full example

# Which scheme to use — a name or "wallpaper"
scheme: gruvbox-dark-hard

prefer_base24: false   # use Base24 variant when available
material_you:  true    # apply HCT color transformation

enabled:
  - fish
  - foot
  - kitty
  - helix
  - sway
  - waybar
  - fuzzel
  - dunst
  - gtk
  - vesktop

font:
  monospace: "JetBrains Mono"
  sansserif: "Ubuntu"
  serif:     "Ubuntu Serif"
  emoji:     "Noto Color Emoji"
  sizes:
    terminal: 12
    desktop:  10
    popups:   10

opacity:
  terminal:     0.95
  applications: 1.0
  desktop:      1.0
  popups:       0.95

Options

scheme string required
The name of a Base16/Base24 scheme, matched case-insensitively against files in ~/.config/coat/schemes/. Use coat list or coat search <term> to browse.

Set to "wallpaper" to extract colors from your current wallpaper on every coat apply run (requires swww).
prefer_base24 bool default: false
When true, use the Base24 variant of the scheme (24 colors instead of 16) if one exists. Falls back to Base16 if no Base24 variant is available.
material_you bool default: false
When true, transforms colors using the HCT color space before applying. Accent colors become more vibrant and harmonized; neutrals gain a subtle tint. See Wallpaper & Material You for the full breakdown.
enabled string[] required
List of modules to theme. Only apps in this list are written to. See Modules for all available values and activation steps.

Base16 color slots

coat's output is always a Base16 palette. Understanding the slot semantics helps when reading module output files or writing your own.

SlotRole
base00Default background
base01Lighter background (status bars, line numbers)
base02Selection background
base03Comments, invisibles, line highlights
base04Dark foreground (status bars)
base05Default foreground, caret, delimiters
base06Light foreground
base07Light background, highlights
base08Red — variables, XML tags, markup link text
base09Orange — integers, booleans, constants
base0AYellow — classes, search text background
base0BGreen — strings, inherited class, markup code
base0CCyan — support, regex, escape chars
base0DBlue — functions, methods, attribute IDs, headings
base0EMagenta — keywords, storage, selector, markup italic
base0FBrown — deprecated, opening/closing embedded language tags
