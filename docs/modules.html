Modules

Activation steps for all 28 supported applications. Click any module to expand it.

Terminals

fish Fish shell — syntax highlighting & pager colors ▸

Writes ~/.config/fish/themes/coat.theme.

One-time activation:

fish_config theme save coat

Permanent (add to ~/.config/fish/config.fish):

fish_config theme choose coat
foot Foot terminal — colors, font, opacity ▸

Writes ~/.config/foot/coat-theme.ini.

Permanent activation (add to ~/.config/foot/foot.ini):

[main]
include=~/.config/foot/coat-theme.ini

Restart foot or open a new window to pick up changes.

kitty Kitty terminal — colors & theme ▸

Writes ~/.config/kitty/coat-theme.conf.

Permanent activation (add to ~/.config/kitty/kitty.conf):

include coat-theme.conf

Reload with Ctrl+Shift+F5 or restart kitty.

tty Linux TTY — console colors via escape sequences ▸

Writes a shell script to ~/.config/coat/tty-colors.sh that sets the 16 terminal palette using escape sequences.

Activation (add to ~/.profile or ~/.bashrc):

[ -f ~/.config/coat/tty-colors.sh ] && source ~/.config/coat/tty-colors.sh

Only takes effect on a real TTY; has no effect inside terminal emulators.

Editors

helix Helix editor — full syntax theme ▸

Writes ~/.config/helix/themes/coat.toml.

Permanent activation (add to ~/.config/helix/config.toml):

[editor]
theme = "coat"

Reload with :config-reload or restart helix.

vscode Visual Studio Code — color theme extension ▸

Writes a complete VS Code theme extension to ~/.vscode/extensions/coat-theme/.

Activation:

  1. Open VS Code → Ctrl+Shift+P → Color Theme
  2. Select Coat

Re-run coat apply vscode and select again after changing schemes.

Window Managers & Compositors

hyprland Hyprland — Base16 color variables ▸

Writes ~/.config/hypr/coat-theme.conf with $base00–$base0F variables.

Permanent activation (add to ~/.config/hypr/hyprland.conf):

source = ~/.config/hypr/coat-theme.conf

Then use the variables anywhere in your config:

general {
    col.active_border = $base0D $base0C 45deg
    col.inactive_border = $base01
}

Reload with hyprctl reload.

i3 i3 window manager — window & bar colors ▸

Writes ~/.config/i3/coat-theme.conf. i3-msg reload is run automatically.

Permanent activation (add to ~/.config/i3/config):

include coat-theme.conf

Or reload manually with $mod+Shift+R.

sway Sway — window borders, client colors, swaybar ▸

Writes two files:

  • ~/.config/sway/coat-theme — window client colors
  • ~/.config/sway/coat-swaybar-theme — complete bar { } block

Permanent activation (add to ~/.config/sway/config):

include ~/.config/sway/coat-theme
include ~/.config/sway/coat-swaybar-theme
The swaybar file is a complete bar { } block. Either remove your existing bar block or manually extract only the colors { } section.

Reload: swaymsg reload

niri Niri compositor — focus ring, borders, tab indicator ▸

Writes ~/.config/niri/coat-theme.kdl.

Permanent activation (add to ~/.config/niri/config.kdl):

include "coat-theme.kdl"

Niri reloads automatically when you save the config.

labwc Labwc — window borders, titlebar, menu ▸

Writes ~/.config/labwc/themerc and runs labwc --reconfigure automatically. No manual activation needed.

Bars & Screen Locker

waybar Waybar — CSS theme with module-specific styling ▸

Writes ~/.config/waybar/coat-theme.css with @define-color variables for all 16 Base16 colors, plus styles for workspaces, clock, battery, network, audio, CPU/memory, tray, MPD, and more.

Permanent activation (add to ~/.config/waybar/style.css):

@import "coat-theme.css";

Reload: killall -SIGUSR2 waybar

The CSS variables (@base00–@base0F) are available for use in your own style.css.

swaylock Swaylock — full lock screen config ▸

Writes ~/.config/swaylock/config directly. No include needed — the theme takes effect the next time you run swaylock.

  • Ring, inside, text, line colors per state (idle, cleared, verifying, wrong, caps-lock)
  • Key highlight and backspace highlight colors
  • Separator and layout indicator colors
  • Opacity from opacity.popups

Launchers

fuzzel Fuzzel — full config including layout settings ▸

Writes the entire ~/.config/fuzzel/fuzzel.ini. No include needed — just launch fuzzel and it will use the new config.

coat overwrites fuzzel.ini completely. Any manual edits to that file will be lost on the next coat apply.

Uses font.sizes.popups for font size.

rofi Rofi — application launcher theme ▸

Writes ~/.config/rofi/coat.rasi.

Permanent activation (add to ~/.config/rofi/config.rasi):

@import "coat.rasi"
bemenu Bemenu — dynamic menu theme via env vars ▸

Writes a shell script to ~/.config/coat/bemenu-theme.sh.

Activation (add to your shell config):

source ~/.config/coat/bemenu-theme.sh

Or source it in your bemenu launcher script directly.

Notifications

dunst Dunst notification daemon ▸

Writes ~/.config/dunst/dunstrc and restarts dunst automatically.

If dunst doesn't restart:

killall dunst && dunst &
  • Background, foreground for all urgency levels
  • Frame color: low=cyan, normal=blue, critical=red
  • Progress bar styling, font from coat.yaml
mako Mako notification daemon ▸

Writes the entire ~/.config/mako/config and runs makoctl reload automatically.

If mako doesn't reload: makoctl reload

  • Background, text, border, progress bar colors
  • Urgency-level overrides (low/normal/high)
  • Font from coat.yaml, layout defaults (margin, padding, border-radius)
avizo Avizo overlay notifications ▸

Writes ~/.config/avizo/config.ini.

System Theming

gtk GTK 3.0 & 4.0 — all GTK applications ▸

Writes to ~/.config/gtk-3.0/gtk.css and ~/.config/gtk-4.0/gtk.css. Uses Libadwaita-compatible color definitions.

Restart GTK applications to apply. Some apps (especially GNOME) may need a full logout/login.

qt Qt5 / Qt6 via qt5ct / qt6ct ▸

Writes color schemes to:

  • ~/.config/qt5ct/colors/coat.conf
  • ~/.config/qt6ct/colors/coat.conf

Also updates the main qt5ct/qt6ct config to select the coat palette.

Requirements: qt5ct and/or qt6ct must be installed.

# Add to ~/.profile or shell config
export QT_QPA_PLATFORMTHEME=qt5ct   # or qt6ct
xresources X11 Xresources — URxvt, XTerm, generic X11 ▸

Writes ~/.Xresources and runs xrdb -merge ~/.Xresources automatically.

This overwrites ~/.Xresources. Back up any custom settings first.
  • #define macros for all 16 Base16 colors
  • URxvt color0–color15, background, foreground, cursor
  • XTerm color settings
  • Generic *.color* X11 resources
  • Font settings from coat.yaml

Utilities & Viewers

bat bat syntax highlighter ▸

Writes a theme to ~/.config/bat/themes/coat.tmTheme.

bat cache --build   # rebuild bat's theme cache
# Add to ~/.config/bat/config
--theme="coat"
btop btop++ system monitor ▸

Writes ~/.config/btop/themes/coat.theme.

In btop: press Esc → Options → Color theme → coat.

cava CAVA audio visualizer ▸

Writes color settings to ~/.config/cava/config.

Restart cava to apply. Or send SIGUSR1: pkill -USR1 cava.

ranger Ranger file manager colorscheme ▸

Writes a Python colorscheme to ~/.config/ranger/colorschemes/coat.py.

One-time activation (add to ~/.config/ranger/rc.conf):

set colorscheme coat
  • Directories, executables, symlinks, sockets, FIFOs, media files
  • VCS (git) status colors
  • Selected, marked, tagged, cut/copied item highlighting
yazi Yazi file manager theme ▸

Writes ~/.config/yazi/theme.toml.

Yazi picks up the theme automatically on next launch.

zathura Zathura document viewer ▸

Writes color settings to ~/.config/zathura/zathurarc.

Restart zathura to apply.

Other

vesktop Vesktop / Vencord — Discord CSS theme ▸

Writes a CSS theme to whichever of these directories exists:

  • ~/.config/vesktop/themes/coat.theme.css
  • ~/.config/Vencord/themes/coat.theme.css

To activate in Vesktop: Settings → Themes → enable coat

To activate in Vencord: Discord → Vencord settings → Themes → enable coat

Themes virtually all Discord UI surfaces via CSS custom properties mapped to Base16 colors.

conky Conky system monitor ▸

Writes color variables to ~/.config/conky/coat-colors.conf.

Add to your conky.conf:

dofile(os.getenv("HOME") .. "/.config/conky/coat-colors.conf")
mangowc MangoHud / mangowc overlay ▸

Writes color settings to ~/.config/MangoHud/MangoHud.conf.

Changes take effect the next time MangoHud is loaded.

Adding a new module

  1. Create src/modules/myapp.c and include/myapp.h
  2. Implement myapp_apply_theme(const Base16Scheme*, const FontConfig*, const OpacityConfig*)
  3. Add an entry to the app_modules[] table in src/core/main.c
  4. Add src/modules/myapp.c to APP_SRCS in the Makefile
  5. Rebuild: make clean && make
